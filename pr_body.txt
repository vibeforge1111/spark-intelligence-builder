## issue.type / issue.severity / issue.title
- **issue.type:** bug
- **issue.severity:** medium
- **issue.title:** False positive instruction detection in route planning

## issue.actual_behavior
Route planning requests containing phrases like 'stop at the pharmacy' incorrectly trigger the instruction save mechanism because the _INLINE_DIRECTIVE regex pattern matches any line starting with 'stop', 'keep', 'remember', etc.

## issue.expected_behavior
Route planning requests should not trigger instruction saving. Only actual instruction directives like 'always remember this' or 'never forget this' should be matched.

## issue.repro_steps
1. Send a route planning request: 'Pick the best route home. Constraints: I must avoid toll roads, my fuel is nearly empty, and I need to stop at the pharmacy before it closes in 15 minutes'
2. Observe that the instruction save mechanism is triggered incorrectly
3. Send an actual instruction: 'always remember this location'
4. Verify that the instruction is saved correctly

## issue.affected_workflow
Instruction detection in user messages

## evidence.before_after_proof
Screenshots needed: Before screenshot should show route planning request with 'stop at the pharmacy' triggering instruction saving. After screenshot should show route planning request no longer triggering instruction saving while actual instructions still work correctly.

- **evidence.safe_links_only:** true
- **evidence.links:** https://github.com/vibeforge1111/spark-intelligence-builder/pull/98
- **evidence.forbidden:** raw secrets, raw logs, raw conversations, private chat IDs, session tokens, cookies, private repo maps, raw memory dumps, full compile JSON, scoring details

## Root cause and fix summary
The _INLINE_DIRECTIVE regex pattern was too broad and matched any line starting with 'stop', 'keep', 'remember', etc., including route planning requests.
The fix refines the regex pattern to only match actual instruction directives with specific patterns: context-aware matching after sentence boundaries, exclusion of pronouns, limited directive set (always, never, stop), and body extraction.

## proposed_fix.approach
Refined the _INLINE_DIRECTIVE regex pattern to only match actual instruction directives with specific patterns: context-aware matching after sentence boundaries, exclusion of pronouns, limited directive set (always, never, stop), and body extraction.

- **proposed_fix.files_expected:** src/spark_intelligence/user_instructions/service.py
- **proposed_fix.tests_or_smoke:** Route planning requests with 'stop at the pharmacy' no longer trigger instruction saving. Actual instruction directives like 'always remember this' or 'never forget this' still work correctly.
