# Agent Landscape Analysis Reference

## Scope

Use this reference when comparing:

- OpenClaw
- Hermes Agent
- Claude Cowork
- adjacent persistent-agent systems

## Evidence Rules

Prefer:

- official docs
- official repos
- official issues
- first-hand architecture or setup docs

Do not present speculation as fact.

## Comparison Categories

Always compare across:

- runtime shape
- onboarding and install
- gateway and adapters
- diagnostics and doctor flows
- scheduler and harness design
- migration/import possibilities
- maintenance burden
- operator trust

## Output Buckets

Always classify findings into:

- borrow
- yoink
- reject
- keep uniquely Spark

## Product Rule

A competitor doing more does not mean we should do more.

A competitor being broader may be a maintenance failure, not a strength.

## Claude Cowork Rule

Treat Claude Cowork primarily as:

- product UX benchmark
- interaction and workflow benchmark

Do not invent internal architecture if it is not publicly available.

## Escalate If

- sources are weak or contradictory
- proprietary internals are not actually public
- the comparison starts drifting into imitation instead of judgment
